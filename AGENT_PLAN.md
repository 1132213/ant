# Ant-Game AI Agent 开发方案与记录

## 开发路线图 (Roadmap)

本项目采用“纯 Agent 结对编程”模式开发，目标是构建一个基于高级规则引擎结合宏观状态机的 AI（后续可结合 MCTS 冲刺上限）。

### Phase 1: 基础设施与信息提取（打基建）
- [x] 创建记录文档 (`AGENT_PLAN.md`)
- [x] 阅读 `SDK/utils` 和 `SDK/backend` 提取关键 API
- [x] 编写 `AI/custom_utils.py`，提供高层状态查询接口（安全建塔点、敌方兵线深度、塔性价比评估等）
- [x] 创建 `AI/ai_custom.py` 的基础骨架

### Phase 2: 建立 Baseline (贪心状态机)
- [x] 基于 `BaseAgent` 编写一个能跑通完整游戏流程的 AI
- [x] 硬编码基础战术：前期攒钱升基地 -> 中期在必经之路建 `Basic` 塔 -> 有钱升 `Heavy` 塔 -> 基地遇袭放超级武器
- [x] 运行本地对战验证胜率
- [x] **流派选择**：选项 C (均衡推进流)
- [x] **状态评估**：通过 `evaluate_frontline_status` 计算双方兵线压迫感，判定当前是 `DEFEND`、`BALANCED` 还是 `ATTACK`
- [x] **动态建塔**：处于防守时在后方建塔，处于进攻时在前线封门建塔
- [x] **经济利用**：余钱充足时主动升级重型防御塔 (Heavy)
- [x] **危机处理**：基地极度危险时自动寻找敌方密集区域释放闪电风暴

### Phase 3: MCTS 战术剪枝与升华 (冲刺上限)
- [x] 放弃繁琐的硬编码，转为引入官方 `PriorGuidedMCTS` 框架。
- [x] 编写 `ai_mcts_custom.py`，将 Phase 2 的启发式规则改造为**动作裁剪器 (Action Pruner)**。
- [x] 将原本成百上千的合法动作，裁剪为 Top 5 的高价值动作（前线建塔、升 Heavy、放闪电风暴、攒钱），喂给 MCTS 进行未来推演。
- [x] 修改打包脚本，支持打包出 `ai_mcts_custom.zip` 供平台评测。

### Phase 4: 最终调优与扩展 (可选)
- [x] 动态威胁热力图计算 (综合距离、血量、速度评估危险区域，引导闪电风暴、建塔和升塔)
- [x] MCTS 性能调优：引入热力图缓存机制，并实现 Time-bound MCTS 硬性时间截断。修复第一回合由于官方 `_expand()` 耗时过长导致的初始化 TLE。
- [x] MCTS 效果优化：放宽启发式剪枝的动作候选数量（建塔位 3->5，升级目标 1->2）；重写 MCTS 的价值评估函数 (`_heuristic_value`)，加入兵线压制、经济优势、防御塔质量及前线加成等软性收益，使短深度 MCTS 能正确识别优势局势。已验证：MCTS 能够依靠更长远的视野在 456 回合内战胜原版基于贪心的 custom AI。
- [ ] 经济收益最大化模型
- [ ] 针对性防御与反制（如：Ice 塔控场、EMP 压制）
